## 背景：20轮测试暴露的四个问题

v3 端到端测试完成，验证点基本通过，但输出质量有四个集中问题：

1. **诗意感太强**：思维流和世界事件都是文学意识流风格，不像真人
2. **17–19 轮重复**：情绪衰退至趋零后输入退化，各层输出几乎相同
3. **思维流字数不足**：平均低于 100 字，信息密度低
4. **情绪永久陷负**：只有衰退通道，无法回升

## 变更二：情绪参数 → 文字描述（4B）

### 问题根因
LLM 是词频模型，理解不了 `fear=0.34`。当前传的是数值，对输出影响极弱。

### 方案
为 8 个 Plutchik 维度 × 10 个区间（0.0–1.0，步长 0.1）定义**文字描述**，替代数值注入 prompt。

**格式：**
```
主导情绪（恐惧）：相当焦虑，注意力容易被拉走，胸口有些紧
情绪底色：有些沉，心里有口气
```

**规则：**
- 主导维度：使用完整区间描述
- 次要维度（value > 0.15，最多 2 个）：使用简短着色短语
- 全维度 ≤ 0.05：输出"情绪平稳，没有明显波动"
- 负值截断：`max(weight, 0.01)`

**描述草稿**：见 `docs/emotion_descriptors_draft.md`（待用户审核）

**实现位置**：新建 `core/emotion_descriptor.py`

## 变更三：DUTIR 作为情绪校准层（4A）

### 方案
OCC 输出后，使用 DUTIR 词典对世界事件文本做情绪方向检测：

```
WorldEngine 生成事件文本
      ↓
OCC 评价 → Plutchik 向量
      ↓
DUTIR 检测事件文本的情绪主方向
      ↓
一致？→ 保留 OCC 结果
不一致？→ 修正主导维度方向：
          压低 OCC 的错误主导，拉高 DUTIR 指示的维度
          其余维度保留
```

**不一致判定**：主导情绪方向（正向/负向）不同即为不一致。

**实现位置**：在 `core/cognitive_engine.py` 的 `emotion_layer()` 内，OCC 之后新增校准步骤。

## 变更四：Drift 重设计（问题 2）

### 废弃现有机制
删除"连续 N 轮强度变化 < 阈值"的稳定性检测。

### 新触发逻辑
基于 `max(所有情绪维度值)` 的阈值分级：

```python
max_dim = max(emotion.to_dict().values())  # 排除 intensity

if max_dim < 0.3:
    drift_prob = 1.0   # 大量 drift：情绪接近平静，意识自由游荡
elif max_dim < 0.7:
    drift_prob = 0.25  # 少量 drift：情绪中等，偶尔游荡
else:
    drift_prob = 0.0   # 无 drift：情绪强烈，专注于当下
```

### Drift 内容：9 类游荡方向

基于 Killingsworth（2010）、Christoff（2016）、DMN 研究：

| 类别 | 基础权重 | 心理依据 |
|------|---------|---------|
| 白日梦/欲望幻想 | 0.20 | 无情绪约束时 DMN 最自然方向 |
| 哲学/存在性探讨 | 0.18 | 平静才有余力思考意义 |
| 创意/审美联想 | 0.15 | 低压时审美感知更活跃 |
| 未来规划/预期想象 | 0.12 | 前瞻性思维是 mind-wandering 最高频类型 |
| 正向记忆回溯 | 0.12 | 放松时正向记忆浮现 |
| 假设社交场景 | 0.10 | 平和状态下轻松社交想象 |
| 自我评估 | 0.07 | 空白状态也有，但低 |
| 反事实思考 | 0.04 | 无情绪触发时很少发生 |
| 反刍 | 0.02 | 无情绪时不容易陷入 |

**情绪乘数矩阵**（最终权重 = 基础权重 + Σ(维度值 × 乘数)，截断至 0.01，归一化）：

```
           反刍  正向回溯  未来规划  反事实  社交演练  白日梦  自我评估  哲学  创意审美
anger      +1.2  -0.3     +1.0     +2.5    +2.5     -0.5    +1.0    +0.8  -0.3
fear       +1.5  -0.3     +2.0     +1.5    +2.0     -0.3    +1.2    +0.7  -0.5
joy        -1.0  +2.5     +1.2     -0.5    +1.0     +2.0    +0.3    +0.5  +2.0
sadness    +2.0  +1.8     -0.5     +0.8    +0.5     +0.8    +1.5    +1.5  -0.3
surprise   +0.3  +0.3     +1.2     +2.0    +1.0     +0.5    +1.5    +1.5  +0.7
disgust    +1.0  -0.5     +0.3     +1.2    +0.8     -0.5    +2.0    +1.5  -0.5
anticipat  -0.5  +0.5     +2.5     +0.3    +1.0     +2.0    +0.3    +0.5  +1.0
trust      -0.8  +1.5     +0.8     -0.5    +1.8     +1.5    +0.3    +0.5  +1.5
```

**林晓雨角色修正系数**（乘在基础权重上）：

| 类别 | 系数 | 理由 |
|------|------|------|
| 反刍 | ×1.4 | 过度自责、习惯独自承受 |
| 社交演练 | ×1.3 | 高度共情、会事后复盘 |
| 创意/审美联想 | ×1.3 | 画画和攒歌单的爱好 |
| 白日梦 | ×1.2 | desires[] 里有明确的克制欲望 |
| 反事实 | ×1.2 | 完美主义者容易"改剧本" |
| 哲学探讨 | ×0.8 | 较少纯粹哲学漫游 |

**WorldEngine 在 drift 时**：只允许 `introspective` / `subtle` 事件，禁止 `dramatic` / `relational`。

## 变更五：Arbiter 全面重构（问题 1 + 问题 3）

### 学术依据
基于 Hurlburt DES 方法论（2008/2013/2015）和 Fernyhough 内语分类学（2015）：

- **内语仅占 ~25%**，其余是视觉意象、无符号思维、身体感知等
- **真实内语是压缩片段**，极少是完整句子
- **对话性结构**：内心有多个"声音"，包括内化他人
- **身体先于语言**（Damasio）：体征 → 未命名感受 → 语言

### 新架构：三阶段生成

```
B1 锚点选择（1次 LLM 调用）
  输入：当前情绪描述 + 处境 + desires + hobbies + 激活记忆 + 感知焦点
  输出：JSON { anchor, anchor_type, trigger }

B2 链条生成（1次 LLM 调用）
  输入：B1 锚点 + 情绪语气参数 + 人物档案
  输出：JSON {
    moments: [{type, content, source?}...],  // 3~5个时刻
    conclusion: "...",  // 可为 null
    write_back: true/false
  }

B3 渲染（确定性代码，0次 LLM 调用）
  按类型规则组装成可读碎片文本
```

### DES 七种思维时刻类型及权重

| 类型 | 权重 | 渲染格式 |
|------|------|---------|
| compressed_speech（压缩内语） | 25% | `道歉——` `他说的那句——` |
| visual_fragment（视觉意象） | 20% | 直接输出，无标记 |
| unsymbolized（无符号思维） | 20% | `〔知道要处理X，说不清〕` |
| body_sensation（身体感知） | 15% | 直接输出 |
| intrusion（侵入性想法） | 10% | 直接输出，突兀 |
| voice_intrusion（他人声音侵入） | 8% | `"陈总说的话"` |
| expanded_speech（展开内语） | 7% | 完整句，偶尔出现 |

### Write-back 准入规则（B2 中由 LLM 判断）

**写入条件（满足其一）：**
- 角色做出了具体的微决定（去哪、做什么、不做什么）
- 解开了一个之前反复循环的问题
- 对某段关系或自我有了新的认识

**不写入条件：**
- `conclusion == null`
- 结论是泛化情绪陈述（"感觉好了一点"）
- 与 memories 中现有内容高度重复

### Profile Write-back 流程

- 每轮 B2 输出的 `(conclusion, write_back)` 暂存在 `tick_store`
- 每 **5 轮**由独立 LLM 调用审查：从 5 个候选中选出值得写入的，去重合并，追加到 `profile.memories`

### 目标字数

思维流目标：**200字**（`_SYS_ARBITER` 指令调整）

## 测试验证点

运行 20 轮后检查：

1. 思维流是否有具体人名/物品/事件？（每 3 轮至少 1 轮）
2. 是否没有感官意象叠加类修辞？（0 容忍）
3. 17–19 轮是否各有新内容？
4. drift 轮次的 9 类方向是否有分布？（不集中在反刍）
5. 20 轮内情绪是否有至少 1 次回升（joy 或 trust > 0.15）？
6. write-back 是否有 ≥1 条结论写入 memories？
