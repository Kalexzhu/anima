# 输出质量修复迭代 — 2026-03-25

## 背景
20 轮端到端测试（v3 架构）完成，验证点基本通过，但输出质量有四个集中问题：

---

## 问题诊断与根本原因

### 问题 1：诗意感太强，修辞多，信息少

**现象举例：**
```
火车鸣笛撕开夜色——往南？往北？
背包带勒进肩胛骨，末班车时刻表在视网膜上灼烧。
昌平果园的咖啡馆突然浮起柑橘香，张明发梢沾着槐花瓣。
手机在口袋里震，震成王阿姨介绍的税务局制服纽扣。
```

**根本原因（3层）：**
1. `_SYS_ARBITER` 说"允许：意识流跳跃、联想、细节感"——这是文学体裁指令，模型把它当卡夫卡风格来写
2. 没有明确的禁止列表；没告诉模型感官叠加、意象融合是不允许的
3. WorldEngine 系统 prompt 说"真实有细节"，模型的理解是意象堆砌，而非信息密度

**人类真实思维规律（与 LLM 的区别）：**
- 人脑想的是具体的事：「那个项目要明天交」「妈妈上次说的话很烦」「今晚吃什么」
- 联想跳跃落在**具体名词**上，不是意象（「柑橘香」→ 落不到地的审美体验）
- 情绪是语气/用词的底色，不是思维的主体内容
- 语言碎片化、不流畅、口语化——不会自动补全成完整隐喻

### 问题 2：轮次 17-18-19 思维流几乎相同

**根本原因：输入退化问题（不是随机性问题）**

```
tick    intensity    事件情况
1       0.65         强烈事件
5       0.65^5≈0.16  进入低强度区
17      ≈0.002       接近零
```

- 情绪×0.7^17 → 趋近零 → WorldEngine 只触发 subtle 事件（"平静生活细节"）
- subtle 事件信息量低，OCC 评价结果基本相同
- memory 检索 query 稳定 → 返回相同 fragment
- 每层输入相同 → 每层输出相同

根治方案：加入**恢复型事件**（`recovery` 模式），当连续 N 轮低强度时注入正向/新鲜刺激，打破输入退化循环。

### 问题 3：思维流字数不足（应 200-300 字）

**根本原因：** 模型用诗意压缩代替叙述——3行意象 ≈ 150字信息量。禁止修辞后，相同内容需要更多字来承载，字数问题自然随 prompt 改写解决。

修复：`_SYS_ARBITER` 改为 200~300字；禁止修辞后模型被迫用直白叙述来填充，信息密度自然提升。

### 问题 4：情绪永久陷负，无法回升

**根本原因：情绪只有衰退通道，没有激励通道**

- WorldEngine 在低强度时只触发 subtle（平静细节），没有设计正向事件类型
- `_pick_relationship` 高强度时偏向负向关系；低强度时随机——正向关系（张明）出现概率不够
- 所有事件的 OCC 评价加上角色的认知偏差（灾难化思维、过度自责），中性事件也往往产出负向情绪

现实中人类会有：朋友主动联系、一顿好吃的、小事得到认可、一个愉快的回忆浮现——这些才是情绪恢复的驱动力。

## 变更方案

### 变更 A：`_SYS_ARBITER` 全面重写

**核心原则：** 心理模拟，不是文学创作

**禁止（明确列举）：**
- 隐喻与拟人（「背包带勒进肩胛骨」类）
- 感官意象叠加（「柑橘香 + 槐花瓣」类）
- 情绪陈述为主体（「我好难受」「我真的撑不住了」）
- 修辞性比喻（「X震成Y」类荒诞联想）

**要求（明确列举）：**
- 200~300字，口语，碎，不流畅
- 有具体的人名/地名/物品/时间
- 从一件事联想到另一件具体的事（不是到意象）
- 情绪只影响语气（烦躁、疲惫、平静），不是主体

### 变更 B：Arbiter 发散锚点注入

每轮从 `desires[]` 或 `hobbies[]` 随机取一个作为"本轮发散锚点"，让 arbiter 在某一时刻联想到这个具体欲望/兴趣。这比多 agent 架构简单得多，且能直接提升"有深度的发散性思考"。

### 变更 C：WorldEngine 事件 prompt 去诗化

系统 prompt 改为"新闻简报"风格，明确禁止意象，并给出好/差样例对比。

### 变更 D：WorldEngine 加恢复型事件（`recovery` 模式）

新增 `_consecutive_low_ticks` 计数器：
- 每轮 intensity < 0.2 时递增，否则清零
- 当 `_consecutive_low_ticks >= 4` 时触发 `recovery` 事件
- `recovery` 事件优先选取正向关系（`valence > 0.5`，如张明）
- 事件内容：朋友联系、小确幸、自我肯定的片刻

```
情绪恢复流程：
[高强度负向] → [×0.7衰退] → [低强度区] → [4轮后触发recovery]
                                              → [正向事件]
                                              → [OCC生成joy/trust]
                                              → [强度回升]
```

## 关于多 Agent 思维模块的判断

> 用户问：是否为角色建立多 agent 思维模块系统更好？

**当前判断：暂缓，prompt 工程先走。**

理由：
- 诗意问题是 LLM 默认偏好，不是架构缺陷——加 agent 解决不了 prompt 问题
- "兴趣/欲望有深度发散"可以通过变更 B（锚点注入）实现，成本低
- 多 agent 带来 N 倍 API 延迟（20轮测试会非常慢）
- 等本轮 prompt 迭代稳定后，如果发散深度仍不足，再评估多 agent

**捕获为 TODO：** 多 agent 思维模块（desire_agent / memory_agent / problem_agent）纳入 Phase 2 规划。

1. 思维流是否有具体人名/物品/事件？（至少每 3 轮有 1 轮）

3. 17-19 轮是否各有新内容？
4. 30 轮内情绪是否有至少 1 次回升（joy 或 trust > 0.15）？
5. WorldEngine 事件中是否出现过 recovery 事件（正向刺激）？
